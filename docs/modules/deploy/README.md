# Модуль `deploy`

**Статус:** реализован happy-path (Sprint 1); **sandbox & security реализован (Sprint 4)** — rootless Docker + egress-allowlist + project-GC ([ADR-010](../../adr/ADR-010-build-sandbox-rootless-egress.md)/[ADR-011](../../adr/ADR-011-project-delete-gc.md)), закрыты [TD-001](../../100-known-tech-debt.md#td-001)/[TD-003](../../100-known-tech-debt.md#td-003) (qa 462 passed / coverage 87.81%, reviewer `production_ready: true`). Открытый приёмочный пункт: живой rootless+egress-энфорс на реальном build-хосте (2 real-stack теста skip) → приёмочный пункт Sprint 5. · **Владелец кода:** `app/deploy`

Build & deploy subsystem: песочница сборки (`vite build` недоверенного кода), деплой nginx-контейнера за Traefik (субдомен — dev; **path-based `/s/{site_id}` — prod**, [ADR-017](../../adr/ADR-017-path-based-site-routing.md)), health-check. Живёт в build-воркерах (`queue=build`). **Prod** встраивается в общий edge-Traefik (`corelysite.shop`, сеть `web`), [ADR-018](../../adr/ADR-018-prod-deployment-shared-traefik-cicd.md).

## DoD
- **(S1)** generic nginx + mount + Traefik-лейблы; health 200 → `status=active` → `LIVE`.
- **(S1) Cleanup-before-run:** `docker rm -f site_{subdomain}` перед каждым `docker run` (идемпотентный передеплой, crash-resumable).
- **(S1) Teardown-on-fail:** при фейле deploy/health — `docker rm -f` контейнера + освобождение route → `status=failed` **до** перевода джобы в `FIXING`/`FAILED`. Никаких orphan-контейнеров с `--restart unless-stopped`, отдающих сайт без health-gate.
- **(S1)** Фейл → `failure_log_ref` в S3 → `FIXING`.
- **(S1)** Lifecycle `site_deployments.status` (`building`/`active`/`superseded`/`failed`) и легальные переходы — [03-architecture.md §5](03-architecture.md#5-lifecycle-сайт-деплоя-state-machine-site_deploymentsstatus).
- **(S4) Изолированная сборка:** rootless Docker ([ADR-010](../../adr/ADR-010-build-sandbox-rootless-egress.md)) вместо привилегированного `docker.sock`; `cap-drop ALL`, `no-new-privileges`, `--read-only` rootfs кроме `/workspace`, non-root UID, лимиты `--cpus`/`--memory`/`--pids-limit`/wall-clock, seccomp ([03-architecture.md §1 → Sprint 4](03-architecture.md#1-sandbox-исполнение-недоверенного-кода)).
- **(S4) Egress-allowlist:** build-контейнер в изолированной сети с egress только к npm-registry. Две стороны ([ADR-010 §C-1](../../adr/ADR-010-build-sandbox-rootless-egress.md)): registry-allowlist `NPM_REGISTRY_ALLOWLIST` на egress-proxy + транспорт `BUILD_EGRESS_PROXY_URL`, инжектируемый воркером в build-контейнер как `http_proxy`/`https_proxy` (прямого маршрута к registry в `internal`-сети нет; обязательно при непустом `BUILD_EGRESS_NETWORK`). Запрет private CIDR/cloud-metadata. App-процессы не блокируются.
- **(S4) Project GC** ([ADR-011](../../adr/ADR-011-project-delete-gc.md)): `DELETE /projects/{pid}` → `202` soft-delete + Celery `project.gc` (teardown всех контейнеров/route, volume, S3 всех ревизий, БД-каскад, отмена in-flight → `FAILED(project_deleted)`); идемпотентно; субдомены не реюзаются ([03-architecture.md §6](03-architecture.md#6-gc-при-удалении-проекта-sprint-4--delete-projectsid-adr-011-закрывает-td-003q-deploy-3)).
- **(Prod) Path-based routing** ([ADR-017](../../adr/ADR-017-path-based-site-routing.md)): при `SITE_ROUTING_MODE=path` сайт — `{APPS_DOMAIN}/s/{site_id}` через PathPrefix+StripPrefix; build с `vite --base=/s/{site_id}/`; `live_url=https://{APPS_DOMAIN}/s/{site_id}/`; health по этому пути. lifecycle/GC/rollback переиспользуются без новых статусов ([03-architecture.md §2A](03-architecture.md#2a-path-based-routing-s-site_id-prod--site_routing_modepath-adr-017)). Закрывает [Q-DEPLOY-2](../../99-open-questions.md#q-deploy-2) (wildcard не нужен).
- **(Prod) Shared edge-Traefik** ([ADR-018](../../adr/ADR-018-prod-deployment-shared-traefik-cicd.md)): prod деплой сайтов в сеть `web` (`external:true`), `TRAEFIK_NETWORK=web`, `APPS_DOMAIN=corelysite.shop`; без своего Traefik/SSL. Контракт `docker-compose.prod.yml` + CI/CD — [07-deployment.md → Prod-модель](../../07-deployment.md#prod-модель-shared-traefik-corelysiteshop-adr-018).
