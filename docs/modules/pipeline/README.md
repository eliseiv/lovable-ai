# Модуль `pipeline`

**Статус:** happy-path реализован (Sprint 1); Fixer loop + resilience реализован (Sprint 2); **слой абстракции LLM-провайдера (`anthropic`|`openai`) реализован ([ADR-032](../../adr/ADR-032-llm-provider-abstraction-openai.md))** · **Владелец кода:** `app/pipeline`, `app/workers`

Ядро: state-machine + диспетчер генерации, 4 LLM-агента через **провайдер-агностичный клиент** (фабрика `build_agent_client` по env `LLM_PROVIDER`: Anthropic SDK `ClaudeAgentClient` дефолт / OpenAI Responses API `OpenAIAgentClient`, [ADR-032](../../adr/ADR-032-llm-provider-abstraction-openai.md)), цикл самовосстановления, cost-control. Живёт в LLM-воркерах (`queue=llm`).

## Граница
- Не отдаёт HTTP клиенту: пишет в Postgres + публикует в Redis pub/sub.
- Сборку/деплой делегирует модулю `deploy` (`queue=build`).

## DoD
- Переходы state-machine + диспетчер (crash-resumable).
- Агенты 1–4 на Claude, prompt caching, tiering моделей.
- **Sprint 2:** контракт Agent 4 (вход спека+дерево+failure_log, выход = схема `agent_output` или `unrecoverable`).
- **Sprint 2:** 4 гарда fix-loop (max attempts / budget / wall-clock / no-progress-by-signature) → `FAILED(reason)`.
- **Sprint 2:** разграничение Celery-retry (инфра) vs доменный FIXING (build-fail) ([ADR-006](../../adr/ADR-006-celery-retry-vs-domain-fixing.md)).
- **Sprint 2:** beat sweeper (`AWAITING_CLARIFICATION` TTL) + reconciler (crash-resume застрявших; базовый S2-DoD покрывал `BUILDING/DEPLOYING/FIXING` — **расширено ADR-019 → см. ниже**; канон скоупа — pipeline §E2).
- **Prod-фикс ([ADR-019](../../adr/ADR-019-reconciler-all-active-states-agent-graceful-fail.md)):** reconciler покрывает **все** активные нетерминальные состояния (`CREATED/INTERVIEWING/SPECCING/BUILDING/DEPLOYING/FIXING`, кроме `AWAITING_CLARIFICATION`) с fail-stuck-ветвью (`stuck_timeout`); шаг агента делает graceful-fail в `FAILED(agent_unavailable)` при недоступности LLM (исчерпание ретраев `429`/`5xx` или не-транзиентный `401`/`403`/`400`) — против concurrency-leak зависших джоб.
- **Prod-фикс ([ADR-019 §Fix round 3](../../adr/ADR-019-reconciler-all-active-states-agent-graceful-fail.md#fix-2026-06-04-round-3--fail-fast-preflight-llm-credential-до-sdk-вызова-прод-инцидент)):** **fail-fast preflight LLM-credential** — каждая агент-таска перед SDK-вызовом проверяет непустой `ANTHROPIC_API_KEY`; пусто → `FAILED(agent_unavailable)` за секунды (не reconciler-TTL), version-agnostic (не зависит от типов исключений SDK). Подстраховка: client-side auth-resolution `TypeError` SDK классифицируется как не-транзиентный LLM-сбой. Уровень — **per-job**, сервис стартует без ключа (pipeline §G).
- `job_events` на каждый переход; SSE pub/sub.
