# 06 — Testing Strategy

Команды и coverage gate — в [02-tech-stack.md](02-tech-stack.md). Runner: `pytest`. Gate: `--cov=app --cov-fail-under=80`.

## Пирамида тестов

```
        /\        E2E (мало): промт → LIVE URL на compose-стеке
       /  \
      /----\      Contract (немного): API ↔ iOS, output Agent 3, вебхук Adapty
     /      \
    /--------\    Integration (средне): API-контракты, резюм пайплайна, идемпотентность
   /          \
  /------------\  Unit (много): state-machine переходы, гарды fix-loop, quota-gate, парсинг
```

## Unit

Быстрые, без внешних сервисов (Postgres/Redis/Claude мокаются или in-memory).

- **State-machine переходы:** легальные/нелегальные переходы (`CREATED→INTERVIEWING`, запрет `LIVE→BUILDING` и т.п.), диспетчер выбирает task по `state`.
- **Гарды fix-loop (4 гарда, [03-architecture §C](modules/pipeline/03-architecture.md#c-четыре-гарда-от-бесконечного-цикла-и-runaway-затрат)):** (a) `max_fix_attempts` обрывает на N-й попытке → `FAILED(build_unrecoverable)`, `retry_count` инкрементируется только на применённом патче `FIXING→BUILDING` (нормативно — pipeline §C(a)); (b) `spend_usd>=budget_usd` → `FAILED(budget_exhausted)`, kill перед LLM-вызовом; (c) `now()>=wall_clock_deadline` → `FAILED(wall_clock_exceeded)`; (d) no-progress: совпадение `failure_signature` со **второго** фейла → `FAILED(no_progress)`, первый фейл (NULL-сигнатура) гард не трогает.
- **Сигнатура фейла ([ADR-005](adr/ADR-005-no-progress-failure-signature.md)):** нормализация на фикстурах логов — два прогона одной ошибки с разными путями/таймстампами/PID дают **одну** сигнатуру; две разные ошибки — разные; парсинг шапки `failure_class`.
- **Reason-коды:** каждый путь `FAILED` ставит корректный `failure_reason` (вкл. `fixer_gave_up` при `unrecoverable:true`, `invalid_agent_output` при исчерпании fix-budget на невалидном дереве).
- **Классификация исключений ([ADR-006](adr/ADR-006-celery-retry-vs-domain-fixing.md)):** транзиентные (Docker/network/S3/`429`/5xx) → ретраябельны; доменные (build/health/validation-fail) → НЕ Celery-retry, идут в `FIXING`. Граница в `app/workers/retry_policy.py`.
- **Output Agent 4:** переиспользование схемы `agent_output` (та же валидация, что Agent 3), reserved `.build.json` исключается из дерева, сигнал `unrecoverable`.
- **thinking-mode пер-агентный ([ADR-023 §Decision (2)/(4)](adr/ADR-023-agent3-token-budget-thinking-room.md), [pipeline §Token-бюджет](modules/pipeline/03-architecture.md#token-бюджет-агентов-adr-023)):** `agent_thinking(agent)` возвращает `{"type":"disabled"}` для Agent 3 **и** Agent 4 (обоих режимов — fixer/editor), `{"type":"adaptive"}` для Agent 1/2; собранный kwargs `messages.create` Agent 4 содержит `thinking={"type":"disabled"}` (проверка реального тела запроса, не слепой мок) — против регресса усечения дерева Agent 4 (прод-инцидент 31-мин правки `agent_output_invalid`). `effort` (`AGENT_EFFORT`) не применяется к Agent 3/Agent 4 (thinking disabled).
- **Structured-output агентов ([ADR-020](adr/ADR-020-agent-structured-output-tool-use-tolerant-parse-retry.md), [pipeline §I](modules/pipeline/03-architecture.md#i-надёжный-structured-output-всех-4-агентов)):** **регрессия на fence (обязательна)** — ответ в ` ```json {…} ``` ` (и без language-tag) парсится `extract_json` **без** `ValueError`, шаг агента **не** уходит в `FAILED` (воспроизводит прод-инцидент run1/run4); **repair неэкранированных кавычек (обязательна, [ADR-026](adr/ADR-026-json-quote-escaping-prompt-and-repair-fallback.md))** — реальный кейс прод-бага 2026-06-08 (`{"questions":[{"position":1,"text":"… (e.g., "Where every cup tells a story")?","kind":"free_text"}]}`, неэкранированные внутренние `"`) чинится repair-fallback `extract_json` → структура извлекается без `ValueError`, шаг **не** уходит в `FAILED`; repair строго fallback (валидный JSON идёт прямым `json.loads`, побайтово эквивалентен); узость (trailing comma / одинарные кавычки **не** чинятся, parse_error сохраняется); доменная валидация (§I.2) применяется **после** repair и не маскируется; **промт-инструкция кавычек** — `STRICT_JSON_SUFFIX` (и собранный промт каждого из 4 агентов) содержит нормативную инструкцию про `\"`-экранирование двойных кавычек внутри string values; **contract на параметры запроса (обязателен)** — собранный kwargs `messages.create` агента **НЕ** содержит форсирующего `tool_choice` (`{"type":"tool",...}`/`{"type":"any"}`) одновременно с включённым `thinking` (проверка реального тела запроса, **не** слепой мок SDK; против регресса HTTP-400 / 100%-отказа thinking⊥forced-tool); **текстовый режим** — структура из `block.text` через `extract_json`, system-промт содержит строгую «raw JSON без фенсов»-инструкцию; **bounded retry** parse/schema-фейла до `AGENT_OUTPUT_MAX_RETRIES` → на исчерпании `FAILED(invalid_agent_output)` (Agent 1/2) либо виток `agent_output_invalid`→FIXING/`invalid_agent_output` (Agent 3/4), каждый retry = отдельная запись `llm_usage`, budget/wall-clock-гард перед каждым вызовом; **диагностируемость** — в `job_events.payload` имя агента + текст ошибки + scrubbed усечённый raw-ответ (без секретов); **промт↔валидатор консистентность по single source (обязателен, [pipeline §I.1a/§I.6](modules/pipeline/03-architecture.md#i1a-канонические-output-схемы-агентов--single-source-of-truth-нормативно))** — для КАЖДОГО из 4 агентов: (а) системный промт декларирует канонический top-level ключ (Agent 1 `questions`, Agent 2 `spec_markdown`, Agent 3/4 `files`/`entry`/`build`, Agent 4 — плюс `unrecoverable`) И (б) валидатор читает тот же ключ (закрывает инцидент Agent 2: промт→`spec`/`specification`, валидатор→`spec_markdown`); **валидатор не падает «empty» на канонической форме (страховка)** — подача минимального канонического объекта (`{"spec_markdown":"# x"}` и т.д.) проходит без schema-фейла; **Agent 2 без противоречия в промте** — `agent2_spec_writer.txt` НЕ содержит инструкции вернуть Markdown-текст / «do not wrap it in JSON».
- **LLM-провайдер: абстракция + фабрика ([ADR-032](adr/ADR-032-llm-provider-abstraction-openai.md)):**
  - **фабрика выбора провайдера** — `LLM_PROVIDER=anthropic` → `ClaudeAgentClient`, `=openai` → `OpenAIAgentClient`, иное значение → fail-fast (не молчаливый дефолт); дефолт (ключ не задан) → `anthropic`.
  - **OpenAI usage/cost-маппинг** — `OpenAIAgentClient` собирает `AgentCall` из Responses-`usage`: `input_tokens`→`input_tokens`, `output_tokens`→`output_tokens`, `input_tokens_details.cached_tokens`→`cache_read_tokens`, `cache_write_tokens`=`0`; `cost_usd` считается по OpenAI pricing-таблице ([observability §2.2A](modules/observability/03-architecture.md#22a-openai-pricing-провайдер-openai-adr-032)); неизвестная модель → консервативный fallback (как Anthropic-fallback на Opus).
  - **OpenAI параметры запроса (contract на реальном теле, не слепой мок)** — собранный kwargs `responses.create`/`stream`: `max_output_tokens` = `AGENTn_MAX_TOKENS`; `reasoning.effort` = `OPENAI_AGENT_EFFORT` для Agent 1/2 и `none` для Agent 3/4 ([ADR-032 §2](adr/ADR-032-llm-provider-abstraction-openai.md)); system-промт в `instructions` **без** `cache_control`-блоков (Anthropic-специфика отсутствует).
  - **OpenAI structured-output — единый текстовый путь** — структура извлекается из `AgentCall.text` тем же `extract_json` (fence-снятие + repair [ADR-026](adr/ADR-026-json-quote-escaping-prompt-and-repair-fallback.md) + bounded retry), **без** native `text.format`/`json_schema`; provider-свитч **не меняет** семантику парсинга/валидации (те же `_validate_*`).
  - **retry-классификация исключений OpenAI ([ADR-032 §5](adr/ADR-032-llm-provider-abstraction-openai.md))** — `is_transient`/`is_non_retryable_llm_failure`/`is_llm_failure` распознают исключения **обоих** SDK: `openai.RateLimitError`/`APIConnectionError`/`APITimeoutError`/`InternalServerError`(5xx) → транзиентные; `openai.AuthenticationError`(401)/`PermissionDeniedError`(403)/`BadRequestError`(400) → не-ретраябельные → graceful `FAILED(agent_unavailable)`. **OpenAI-клиент НЕ оборачивает** SDK-исключения (`AsyncOpenAI` в `__init__` без `try/except`; `run_agent` пробрасывает «as-is») — невалидный непустой `OPENAI_API_KEY` даёт **request-time** `openai.AuthenticationError(401)` (не `LLMCredentialError`), уже non-retryable; пустой `OPENAI_API_KEY` отсекается preflight'ом (см. ниже). Контракт обёртки — нормативный single source [ADR-032 §5](adr/ADR-032-llm-provider-abstraction-openai.md) (`LLMCredentialError` остаётся доменным только для anthropic-пути client-side `TypeError`).
  - **preflight `OPENAI_API_KEY` ([ADR-019 §G](adr/ADR-019-reconciler-all-active-states-agent-graceful-fail.md))** — при `LLM_PROVIDER=openai` и пустом/whitespace `OPENAI_API_KEY` агент-таска → fail-fast graceful `FAILED(agent_unavailable)` до SDK-вызова и без инкремента Celery-retry (зеркало anthropic-preflight); какой ключ проверять — по `LLM_PROVIDER`.
  - **Anthropic-путь без регрессий** — при `LLM_PROVIDER=anthropic` (дефолт) собранное тело `messages.stream`, pricing, `LLMCredentialError`, usage-маппинг — байт-в-байт прежние (provider-свитч не затрагивает существующие anthropic-тесты).
- **Read/idle-таймаут LLM-клиента ([ADR-035](adr/ADR-035-llm-stream-read-timeout.md), [pipeline §Агенты → read-timeout](modules/pipeline/03-architecture.md#агенты-anthropic-sdk)):**
  - **(1) таймаут → транзиентный ретрай, НЕ stuck (обязательна)** — повисший stream (мок SSE-итератора, не отдающий чанк дольше `LLM_READ_TIMEOUT_S`) → SDK поднимает `APITimeoutError` → `is_transient(exc)`=`True` → агент-таска уходит в **Celery-ретрай** (не reconciler-`stuck_timeout`, не `FAILED` сразу); исчерпание `max_retries` на `APITimeoutError` → `FAILED(agent_unavailable)` (`is_llm_failure`), **не** `infra_error`/`stuck_timeout`. Проверяется для **обоих** SDK (`anthropic.APITimeoutError` И `openai.APITimeoutError`) — классификатор `retry_policy` уже их содержит (тест фиксирует, что они в `TRANSIENT_EXCEPTIONS`, против регресса удаления).
  - **(2) `httpx.Timeout` собран на клиенте (contract на реальном конструкторе, не слепой мок)** — `AsyncAnthropic`/`AsyncOpenAI` инстанцируются с `timeout=httpx.Timeout(read=settings.llm_read_timeout_s, connect=settings.llm_connect_timeout_s, …)`; `read` берётся из `LLM_READ_TIMEOUT_S`, `connect` из `LLM_CONNECT_TIMEOUT_S` (не хардкод). Это **read-**, не total-таймаут — total-компонент не задаётся конечным значением, ограничивающим суммарную длительность стрима.
  - **(3) межчанковый легитимный стрим не рвётся** — мок-стрим, отдающий чанки с интервалом **< `LLM_READ_TIMEOUT_S`** в течение суммарно дольше `LLM_READ_TIMEOUT_S` (имитация длинного thinking-вывода), завершается **успешно** без `APITimeoutError` (read-таймер сбрасывается на каждом чанке; суммарная длительность не ограничена) — против регресса в total-таймаут.
  - **(4) симметрия обоих SDK** — таймаут-конфиг и поведение (1)–(3) идентичны для `ClaudeAgentClient` и `OpenAIAgentClient` ([ADR-032](adr/ADR-032-llm-provider-abstraction-openai.md)); один read-ключ на оба клиента.
- **Auth (Sprint 3):** Apple identity token verify (mock JWKS) — подпись/`aud`/`iss`/`exp`/`nonce`; индексируемый O(1) token lookup (один argon2-verify, независимость от числа юзеров — [TD-004](100-known-tech-debt.md#td-004)); парсинг `lv_<key_id>_<secret>`; rate-limit token bucket; concurrency cap. Нормативный контракт — [modules/auth/03-architecture.md](modules/auth/03-architecture.md).
- **Image attachments — валидация (sniff/лимиты, [ADR-034](adr/ADR-034-user-image-attachments-vision-site-assets.md)):** детект типа **по magic bytes** на фикстурах (валидные PNG/JPEG/WebP/GIF → принят с корректным `mime`; не-image байты с image-`Content-Type`/именем → `422 unsupported_image_type` — **доверять содержимому, не заголовку**); лимиты `MAX_IMAGES_PER_JOB`/`MAX_IMAGE_BYTES`/`MAX_IMAGES_TOTAL_BYTES`/`MAX_IMAGE_DIMENSION_PX` → `422` с соответствующим `reason` (граничные значения: ровно лимит проходит, +1 байт/файл/px отклоняется).
- **Quota-gate:** активный access level + остаток квоты → пропуск; нет прав / квота исчерпана → `402`.
- **Cost-ledger:** агрегация `llm_usage.cost_usd` в `generation_jobs.spend_usd`.
- **Парсинг/валидация:** схема output Agent 3 (дерево файлов), парсинг вебхука Adapty.

## Integration

С реальными Postgres + Redis (из compose), Claude и Docker-сборка мокаются.

- **API-контракты:** все endpoints из [modules/api/02-api-contracts.md](modules/api/02-api-contracts.md) — статусы (`202`, `402`, RFC-7807), схемы ответов, auth (401 без Bearer).
- **Резюм пайплайна:** `POST /answers` переводит `AWAITING_CLARIFICATION → SPECCING` и ставит задачу; sweeper экспайрит зависшую джобу.
- **FIXING happy-recovery (S2):** `DEPLOYING` build-fail → `FIXING` (deploy-teardown + `failure_log_ref` записан) → `task_fix` → валидный патч Agent 4 → `BUILDING` (пересборка нового source.tgz → dist) → `DEPLOYING` (cleanup-before-run, идемпотентный передеплой того же субдомена) → `LIVE`. `retry_count` инкрементирован ровно один раз (на `FIXING → BUILDING`).
- **FIXING → FAILED по каждому гарду (S2):** интеграционно прогнать исчерпание (a)/(b)/(c)/(d) → терминальный `FAILED` с корректным reason + последний `failure_log` доступен через `GET /jobs/{id}`.
- **Celery retry/backoff (S2):** транзиентный инфра-сбой (мок Docker/S3 timeout) ретраится с backoff и восстанавливается без инкремента `retry_count`; исчерпание `max_retries` на **не-LLM** инфра-сбое → `FAILED(infra_error)`.
- **Sweeper + reconciler (S2):** sweeper переводит просроченный `AWAITING_CLARIFICATION → FAILED(clarification_timeout)`, идемпотентен; reconciler ре-диспетчеризует (ветвь 1, resumable-состояния `BUILDING/DEPLOYING/FIXING`) джобу, застрявшую дольше `STUCK_THRESHOLD_S`, без дабл-постановки таски (`FOR UPDATE SKIP LOCKED` + `dispatch:{job_id}`-lock). Полный скоуп reconciler по всем активным нетерминальным состояниям и ветвь fail-stuck (`stuck_timeout`) для LLM-фазных состояний — строка ниже / [ADR-019](adr/ADR-019-reconciler-all-active-states-agent-graceful-fail.md) (pipeline §E2).
- **Reconciler — все активные состояния + concurrency-leak (прод-фикс, [ADR-019](adr/ADR-019-reconciler-all-active-states-agent-graceful-fail.md)):** (1) джоба, провисевшая в `CREATED`/`INTERVIEWING`/`SPECCING` дольше `STUCK_THRESHOLD_S` без живой таски, переводится reconciler'ом в `FAILED(stuck_timeout)` и **освобождает concurrency-слот** (после этого `POST /projects` того же юзера не получает `402 concurrency_limit`); (2) `AWAITING_CLARIFICATION` reconciler'ом **не** трогается (его экспайрит только sweeper по TTL); (3) reconciler использует `last_transition_at` (не `updated_at`) — апдейт `spend_usd` cost-ledger'ом не сбрасывает heartbeat и не «оживляет» stuck-джобу ложно.
- **Graceful-fail агента при недоступности LLM (прод-фикс, [ADR-019](adr/ADR-019-reconciler-all-active-states-agent-graceful-fail.md)):** (1) транзиентный сбой Claude (мок `429`/`5xx`/timeout) ретраится; исчерпание `max_retries` → `FAILED(agent_unavailable)` (не `infra_error`), слот освобождён; (2) не-транзиентный сбой (мок `AuthenticationError`/`401` — невалиден `ANTHROPIC_API_KEY`) → **немедленно** `FAILED(agent_unavailable)` без сжигания ретраев; (3) джоба **не** остаётся в активном state `INTERVIEWING`/`SPECCING` — терминализируется и не держит слот.
- **Fail-fast preflight пустого `ANTHROPIC_API_KEY` (прод-фикс round 3, [ADR-019 §Fix round 3](adr/ADR-019-reconciler-all-active-states-agent-graceful-fail.md#fix-2026-06-04-round-3--fail-fast-preflight-llm-credential-до-sdk-вызова-прод-инцидент) / pipeline §G):** (1) **integration:** при **пустом/отсутствующем** `ANTHROPIC_API_KEY` `POST /projects` → джоба за **секунды** (не 15 мин) → `FAILED(agent_unavailable)`; `failure_reason` именно `agent_unavailable` (**не** `wall_clock_exceeded`/`stuck_timeout`); слот свободен → следующий `POST /projects` **не** `402`; без необработанного `RuntimeError`/`TypeError` (таска не в «unexpected»); reconciler **не** участвует (переход до `STUCK_THRESHOLD_S`); (2) **unit (preflight):** агент-таска при пустой/whitespace-only `Settings.anthropic_api_key` → graceful `FAILED(agent_unavailable)` без вызова Anthropic SDK и без инкремента Celery-retry; (3) **unit (подстраховка):** инъекция встроенного `TypeError("Could not resolve authentication method...")` из фабрики Anthropic-клиента классифицируется как не-транзиентный LLM-сбой (`agent_unavailable`), не ретраится, не уходит в «unexpected».
- **Инвариант терминальности — race FAILED↔LIVE ([ADR-029](adr/ADR-029-terminal-state-invariant-no-overwrite-deploy-guard-reconciler-revoke.md), [pipeline §Инвариант терминальности](modules/pipeline/03-architecture.md#инвариант-терминальности-состояния-adr-029)), прод-фикс рассинхрона:**
  - **unit/integration (CAS-барьер `transition()`):** при джобе уже в терминале (`FAILED`/`LIVE`) повторный `transition(..., LIVE)` — **no-op**: `state` остаётся `FAILED`, **новый** `job_events` НЕ пишется, publish/push НЕ вызывается, terminal-метрики НЕ дублируются (счётчик `lovable_jobs_total`/`job_failed_total` не инкрементируется второй раз), лог `transition_skip_terminal`. CAS затрагивает 0 строк при терминальном `state`.
  - **integration (reconciler пометил FAILED → `_deploy` НЕ пишет LIVE):** reconciler/`fail_job` перевёл джобу в `FAILED` **во время** живого `_deploy` (мок долгого `docker run`+`wait_until_live`); финальный путь `_deploy` (re-read `job.state` §B + барьер §A) **не** оставляет `LIVE` — итог `FAILED` (терминал, записанный первым, побеждает); deploy-контейнер при необходимости снят teardown-инвариантом; рассинхрона FAILED↔LIVE нет.
  - **unit (reconciler revoke, [ADR-029 §C](adr/ADR-029-terminal-state-invariant-no-overwrite-deploy-guard-reconciler-revoke.md)):** при терминализации reconciler'ом (`stuck_timeout`/`wall_clock_exceeded`) вызывается best-effort `revoke(task_id, terminate=True)` по `task_id` из `job_task:{job_id}`; промах/отсутствие ключа НЕ валит fail-stuck (терминализация доходит).
  - **watchdog НЕ роняет живую прогрессирующую джобу ([ADR-029 §Связь с watchdog](adr/ADR-029-terminal-state-invariant-no-overwrite-deploy-guard-reconciler-revoke.md), [pipeline §E2 heartbeat](modules/pipeline/03-architecture.md#e2-reconciler-застрявших-активных-состояний-crash-resume--concurrency-leak-guard-adr-019)):** distinct failure-event витка (`_handle_invalid_patch`/`fix_rejected` в `FIXING`, без смены `state` и без инкремента `retry_count`) **обновляет** `last_transition_at` (heartbeat) → reconciler НЕ помечает stuck живую прогрессирующую edit/fix-джобу между витками Agent 4; обрыв непродуктивной живой джобы остаётся за wall-clock §C(c) / no-progress §C(d), а не stuck-таймером; cost-ledger (`spend_usd`) heartbeat НЕ трогает.
- **failure_log в S3 (S2):** deploy пишет `logs/{job_id}/build.log` с машинной шапкой; pipeline читает по `failure_log_ref`, парсит `failure_class`, подаёт хвост (`FIXER_LOG_TAIL_BYTES`) в Agent 4.
- **Идемпотентность:** повторный `POST /projects` / `/edits` с тем же `Idempotency-Key` не создаёт дубль; повторный `/answers` на уже продвинувшейся джобе безопасен ([Q-PIPELINE-2](99-open-questions.md#q-pipeline-2)); повторный вебхук с тем же `adapty_event_id` обрабатывается один раз (включая **token-grant**: кредиты не начисляются повторно, [ADR-027 §E](adr/ADR-027-adapty-webhook-bearer-token-grant.md)).
- **Image attachments — приём, инжект, vision, GC ([ADR-034](adr/ADR-034-user-image-attachments-vision-site-assets.md)):**
  - **multipart-приём:** `POST /projects` и `/edits` как `multipart/form-data` (`prompt`/`instruction` Form-поле + `images` файлы) → `202`; строки `attachments` созданы (`project_id`/`job_id`/`s3_ref`/`mime`/`size_bytes`), S3-объект под `uploads/{project_id}/{att_id}.{ext}` записан; запрос без `images` → прежнее поведение (нулевые регрессии). `Idempotency-Key` обязателен (как прежде).
  - **идемпотентность приёма:** повтор того же `Idempotency-Key` с теми же файлами → **не дублирует** строки `attachments`/S3-объекты (replay возвращает ту же джобу).
  - **vision-вход обоих провайдеров (мок SDK, contract на реальном теле):** при непустом `images` собранное тело Anthropic несёт image content-блоки (`type:"image"`, base64) **+** text-блок; OpenAI — `input_image` (data-URL) **+** `input_text`; для **Agent 3 (Builder) image-блоки НЕ передаются** ([ADR-034 §D3](adr/ADR-034-user-image-attachments-vision-site-assets.md)); дефолт `images=None` → тело байт-в-байт прежнее (текстовый путь, нулевые регрессии); `extract_json`-выход не меняется.
  - **детерминированный инжект:** воркер на фазе build материализует **ВСЕ** фото проекта (`attachments WHERE project_id`) как `public/uploads/{att_id}.{ext}` поверх дерева Agent 3 (после валидации `agent_output`, до `pack_source_tgz`); путь относительный/детерминированный; файл инжекта присутствует в `source.tgz`; фото с генерации доступно на последующей правке (скоуп `project_id`, не джобы).
  - **GC префикса ([ADR-011](adr/ADR-011-project-delete-gc.md)):** `project.gc` удаляет S3-префикс `uploads/{project_id}/` (отдельным `delete_prefix`) **и** строки `attachments` проекта; FK-порядок hard-delete (`attachments` до `generation_jobs`/`projects`) не нарушается; идемпотентно (повторный GC — no-op).
- **Crash-resumability:** джоба в промежуточном `state` подхватывается диспетчером после рестарта воркера.
- **Non-transactional DDL-миграции — РЕАЛЬНОЕ применение DDL (обязателен, [ADR-031](adr/ADR-031-alembic-sync-engine-non-transactional-ddl.md), [03-data-model → Migration-guidance](03-data-model.md#migration-guidance--non-transactional-ddl-adr-031)):** на свежей эфемерной БД прогон `alembic upgrade head` через **тот же `env.py` / тот же URL-механизм, что прод-migrate** (sync psycopg, `DATABASE_URL_SYNC`) обязан после апгрейда проверить, что значение enum **реально создано в `pg_enum`**:
  ```sql
  SELECT 1 FROM pg_enum e JOIN pg_type t ON e.enumtypid = t.oid
  WHERE t.typname = 'job_state' AND e.enumlabel = 'EDITING';
  ```
  — а не только что `alembic_version` встал на `20260612_0001`. ⚠️ **Проверка одного `alembic_version` НЕДОСТАТОЧНА** — именно она пропустила прод-инцидент 2026-06-12 (version закоммитился, `ADD VALUE` не применился под async-движком). Тест ОБЯЗАН гонять прод-путь (`env.py`), а не отдельный sync-коннект мимо него. **Идемпотентность ([ADR-031 §C](adr/ADR-031-alembic-sync-engine-non-transactional-ddl.md)):** повторный прогон миграции `20260612_0001` на БД, где `EDITING` **уже** существует (эмуляция 4 исправленных вручную прод-БД), — **не падает** (`ADD VALUE IF NOT EXISTS` / проверка `pg_enum`) и не дублирует значение.

## Contract

- **iOS ↔ API:** OpenAPI-схема FastAPI соответствует контракту в `modules/api`. Pact/schema-snapshot.
- **Output Agent 3:** строгая валидируемая схема дерева файлов ([Q-PIPELINE-1](99-open-questions.md#q-pipeline-1)) — фикстуры валидных/невалидных деревьев.
- **Adapty webhook ([ADR-027](adr/ADR-027-adapty-webhook-bearer-token-grant.md)):** фикстуры реальных payload'ов событий → корректный апдейт `subscriptions`/`billing_events`. Нормативный чек-лист сценариев ([02-api-contracts §1](modules/billing/02-api-contracts.md#коды-ответов-и-always-200-on-bad-input-adr-027-b)):
  - **(a) Bearer-авторизация:** нет/неверный `Authorization: Bearer` → `401`; верный → проходит. Сравнение constant-time.
  - **(b) Пустой `ADAPTY_WEBHOOK_SECRET` → `500`** (мисконфигурация), **до** парсинга тела.
  - **(c) Always-200-on-bad-input** (после валидной авторизации, НИКОГДА не `5xx`): пустое тело → `200 ignored:empty_body`; не-JSON → `invalid_json`; не объект → `not_an_object`; нет `event_id` → `missing_event_id`; неизвестный `event_type` → `200 ignored:{event_type}`; нет/не-маппится `customer_user_id` → `missing_customer_user_id` (+ `billing_events user_id=NULL`).
  - **(d) Валидное событие** `subscription_started`/`renewed` → `200 applied`: `subscriptions` обновлён И `bonus_generations_balance += tier_tokens` по тиру `vendor_product_id` (WEEKLY/YEARLY/fallback GRANT); `credit_grants(created_by='adapty', idempotency_key=event_id)` записан в той же транзакции.
  - **(e) Повтор `event_id`** → `200 duplicate`, токены **не** начислены второй раз (UNIQUE `adapty_event_id`).
  - **Дефенсивный парсинг:** `event_id` из `id`; `customer_user_id` из `profile.customer_user_id`/`user_id`; `vendor_product_id` из `event_properties.*`/`product_id`; `event_type` нормализуется `.lower()`.
  - **`subscription_cancelled`** → `200 applied`/`ignored` по модели §2.3 (status сохраняется, токены НЕ трогаются); `subscription_expired`/`refunded`/`billing_issue_detected` — без начисления токенов.
  - **`5xx` только на реальный сбой БД** (мок недоступности БД при коммите) → строка `billing_events` остаётся `processed_at IS NULL`.
- **Чистота публичной OpenAPI-схемы (прод, тема B — [modules/api/02-api-contracts.md → Публичная API-документация](modules/api/02-api-contracts.md#публичная-api-документация-swaggeropenapi--нормативный-стандарт)):** тест сериализует `app.openapi()` и ассертит **отсутствие** запрещённых подстрок (case-insensitive): `Sprint`/`Спринт`, `ADR-`, `TD-`, `Interviewer`/`Builder`/`Fixer`/`Spec writer`/`Agent 1..4`, `reconciler`/`sweeper`/`dispatcher` (нормативный grep-чек-лист B.7). Наличие любой → **major**. Дополнительно: `/metrics`/`/healthz`/`/readyz` отсутствуют в схеме (`include_in_schema=False`); webhook Adapty присутствует с пометкой S2S; глобальные `title`/`description` на русском без внутренних маркеров; `tags` соответствуют доменному перечню B.4.
- **Async-код из синхронной Celery-задачи (прод-фикс баг A2 — [observability §7](modules/observability/03-architecture.md#7-async-выполнение-async-кода-из-синхронной-celery-задачи-нормативный-паттерн-loopengine)):** `metrics.refresh` (и аналогичные async-Celery-задачи) прогоняются **≥2 раз подряд в одном процессе воркера** (с реальным Postgres/asyncpg) — оба прогона успешны, **нет** `RuntimeError: Future attached to a different loop`, нет утечки asyncpg-соединений между прогонами; Gauge'и метрик наполняются на каждом прогоне.
- **Детерминизм локализации ([ADR-028](adr/ADR-028-deterministic-source-prompt-language-detection.md) ревизует [ADR-025](adr/ADR-025-content-language-autodetect-spec-marker.md), прод-фикс недетерминизма):** серверный детект языка из исходного промпта **детерминирован и не зависит от LLM** — русский (кириллица) промпт → `ru` **на каждом прогоне** (не флапает); английский → `en`; смешанный/неуверенный script → fallback `en`; серверная language-директива инжектируется в Agent 1/Agent 2 (не само-детект); маркер `**Content language:**` несёт значение `content_language`. Нормативный чек-лист сценариев (unit-детерминизм детекта, fallback, script-маппинг, contract-инжекция директивы, live двуязычная E2E с акцентом на детерминизм) — единый источник [pipeline §Язык/локализация → Критерии приёмки](modules/pipeline/03-architecture.md#языклокализация-контента-сайта--детерминированный-детект-adr-028-ревизует-adr-025); здесь — ссылка, не дублируем.

## E2E (happy-path, главный для Sprint 1)

На dev-стеке (compose в WSL2), реальный Claude, реальный `vite build`:

```
curl POST /v1/projects {промт} → job_id
poll GET /jobs/{id} → AWAITING_CLARIFICATION
GET /jobs/{id}/questions
POST /jobs/{id}/answers
poll GET /jobs/{id} → LIVE + live_url
GET {live_url} → HTTP 200, реально собранный Vite-сайт
```

Health-check сайта = 200 — критерий прохождения.

### Image attachments E2E (фото-на-сайте, [ADR-034](adr/ADR-034-user-image-attachments-vision-site-assets.md))

- **Фото-на-сайте (live):** `POST /v1/projects` multipart с `prompt` + реальным изображением → poll → `LIVE`; `GET {live_url}/uploads/{att_id}.{ext}` отдаёт **тот же файл байт-в-байт** (HTTP 200, image MIME) — ассет реально на собранном сайте через `public/uploads/*` → `dist/uploads/*` → path-routing `{APPS_DOMAIN}/s/{site_id}/uploads/...`. Тот же сценарий для `/edits` (новое фото на правке).
- **Vision × thinking (Anthropic) — staging-проверка ДО раскатки ([Q-IMG-1](99-open-questions.md#q-img-1)):** реальный Anthropic-вызов Agent 1/2 с `thinking=adaptive` **+** image-блоком возвращает **200** (не HTTP 400 thinking⊥image). **Открытый приёмочный пункт** — блокирует раскатку vision на Anthropic-инстансы (при 400 — развязка per-agent thinking-disable для vision-вызовов, [ADR-034 §Главный риск](adr/ADR-034-user-image-attachments-vision-site-assets.md)). OpenAI-путь (`reasoning` + `input_image`) риска не несёт.

### Path-based routing (prod, `SITE_ROUTING_MODE=path`, [ADR-017](adr/ADR-017-path-based-site-routing.md))

Что тестировать при path-режиме (в дополнение к субдомен-E2E):
- **Traefik-labels:** в режиме `path` навешиваются `PathPrefix("/s/{site_id}")` + StripPrefix-middleware `/s/{site_id}` + `entrypoints=websecure` (unit — генерация лейблов по `SITE_ROUTING_MODE`; assert на имена router/middleware/service и `prefixes`).
- **StripPrefix:** запрос `GET {APPS_DOMAIN}/s/{site_id}/` доходит до nginx как `/` (integration с поднятым Traefik+nginx, либо unit на конфиг middleware); без StripPrefix nginx получает `/s/{site_id}` → 404.
- **Нормализация build-команды (нормативно, [ADR-017 §Fix 2026-06-08](adr/ADR-017-path-based-site-routing.md#fix-2026-06-08--vite-not-found--потеря---base-через-npm-run-build-прод-инцидент)):** воркер приводит `build.command` к `npx vite build` (unit-таблица входов: голый `vite build` → `npx vite build`; `npm run build` → `npx vite build`; `npx vite build` → без изменений). В режиме `path` `--base=/s/{site_id}/` инжектится **в токен `npx vite build`** (assert: итог `npm install && npx vite build --base=/s/{site_id}/`, а **не** `npm run build --base=...` в хвосте).
- **Vite base-path (нормативно):** build-команда в режиме `path` содержит `--base=/s/{site_id}/`, доходящий до vite (unit на формирование argv build-контейнера); собранный `dist/index.html` ссылается на ассеты как `/s/{site_id}/assets/...` (**не** `/assets/...`), и `GET {live_url}assets/...` отдаёт 200 (E2E реальной сборкой) — без base-path ассеты 404 за StripPrefix → пустой экран (прод-инцидент 2026-06-08).
- **`live_url`:** формат `https://{APPS_DOMAIN}/s/{site_id}/` (со слешем); health-check цель = `live_url`.
- **Режим `subdomain` не сломан:** дефолтный dev-режим (Host-router, `apps.localhost`) проходит существующий E2E без изменений.

### OpenAI-провайдер E2E ([ADR-032](adr/ADR-032-llm-provider-abstraction-openai.md))

- **Provider-свитч E2E (real-stack, при наличии `OPENAI_API_KEY`):** тот же happy-path E2E (промт → `LIVE` + `live_url`, health 200) на стеке с `LLM_PROVIDER=openai` + GPT-5-class `AGENTn_MODEL` — подтверждает, что слой агентов провайдер-агностичен (тот же пайплайн на OpenAI). Где нет `OPENAI_API_KEY` — **живой приёмочный пункт** (как anthropic real-stack E2E): мок OpenAI Responses-клиента с фикстурами output агентов.
- **Усечение дерева Agent 3/4 на OpenAI (приёмочный, риск из [ADR-032 Consequences](adr/ADR-032-llm-provider-abstraction-openai.md)):** генерация **сложного** сайта на `LLM_PROVIDER=openai` — Agent 3 (Builder) с `reasoning.effort=none` + `max_output_tokens=AGENT3_MAX_TOKENS` отдаёт **полное** валидное дерево (не усечён reasoning-токенами); зеркало anthropic-проверки [ADR-023](adr/ADR-023-agent3-token-budget-thinking-room.md) (ни один cap не усекает вывод). Где нет живого OpenAI — отмечается как live-приёмочный пункт.

### Prod-deploy контракт ([ADR-018](adr/ADR-018-prod-deployment-shared-traefik-cicd.md))

- **`docker-compose.prod.yml`:** `api` — `expose` без `ports:` (нет 80/443 наружу); подключён к `web` (`external: true`) + `default`; labels `Host("corelysite.shop")`+`websecure`; нет своего `traefik`/ACME-сервиса; Postgres/Redis/MinIO без `ports:`. Env-ключи символ-в-символ ([07-deployment env](07-deployment.md#канонический-список-ключей)) — devops-reviewer.
- **CI/CD:** `deploy` job имеет `needs: [lint, type-check, test]` (деплой только на зелёном); prod GitHub Secrets соответствуют [05-security → Секреты](05-security.md#секреты). Живой SSH-деплой на сервер — открытый приёмочный пункт (живой стек); автоматизируемое — проверка структуры workflow (`needs:`, шаги).
