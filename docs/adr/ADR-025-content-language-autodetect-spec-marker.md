# ADR-025 — Язык контента сгенерированного сайта: авто-детект из текста пользователя через spec-маркер `**Content language:**`

**Статус:** Accepted (ревизован [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md) — механизм определения языка заменён: детерминированный серверный script-детект из ИСХОДНОГО промпта + серверная инжекция директивы; приоритет → ПРОМПТ, не ответы; добавлено поле `generation_jobs.content_language`. Маркер `**Content language:**` и downstream-роль Agent 3/4 сохранены.)
**Дата:** 2026-06-08
**Связано:** [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md) (ревизия механизма), [ADR-020](ADR-020-agent-structured-output-tool-use-tolerant-parse-retry.md) (structured-output агентов, канонические output-схемы §I.1a), [pipeline §03 → Язык/локализация](../modules/pipeline/03-architecture.md#языклокализация-контента-сайта--детерминированный-детект-adr-028-ревизует-adr-025), [08-product-decisions §Локализация](../08-product-decisions.md), [Q-LOCALE-1](../99-open-questions.md#q-locale-1)

> **РЕВИЗИЯ [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md) (2026-06-11).** Прод-баг: само-детект языка LLM (Agent 1/Agent 2) недетерминирован — один русский промпт давал то русские, то английские вопросы. ADR-028 заменяет §Decision-механику: язык определяется **детерминированно на сервере** из **исходного промпта** (script-эвристика) **до** Agent 1, сохраняется в `generation_jobs.content_language`, инжектируется сервером в директиву Agent 1/Agent 2; маркер `**Content language:**` берёт значение из серверного детекта; **приоритет — ПРОМПТ** (не ответы). Ниже **Decision п.1, п.2 (приоритет), п.5 (вопросы Agent 1)** действуют в редакции ADR-028; маркер, Agent 3 (`<html lang>`+контент), Agent 4 (сохранение языка), убранная кириллица из промтов (п.3, п.4, п.6) — **в силе без изменений**. Нормативная механика — [pipeline §03 → Язык/локализация](../modules/pipeline/03-architecture.md#языклокализация-контента-сайта--детерминированный-детект-adr-028-ревизует-adr-025).

## Context

Прод-баг: сгенерированный сайт выходит на **русском**, хотя пользователь писал промпт и отвечал на уточняющие вопросы на **английском**. Диагностика по коду выявила три корня:

1. **Главный.** Эталонные примеры в системных промтах агентов содержат **кириллицу и локаль-специфичные термины**: `app/pipeline/prompts/agent2_spec_writer.txt` несёт русский пример output (`{"spec_markdown":"# Спецификация\n…"}`) и термин `(ТЗ)`; те же `(ТЗ)` — в `agent4_fixer.txt` и `agent4_editor.txt`. Few-shot-пример на русском смещает язык генерации: Claude копирует язык примера → Agent 2 пишет `spec_markdown` по-русски → Agent 3 строит сайт из русской спеки → сайт русский.
2. Ни один из 4 системных промтов **не инструктирует** про язык user-facing контента.
3. Язык пользователя **нигде не определяется и не фиксируется**; Agent 3 не выставляет корневой `<html lang>`.

Минимальная output-схема Agent 2 в [pipeline §I.1a](../modules/pipeline/03-architecture.md#i1a-канонические-output-схемы-агентов--single-source-of-truth-нормативно) сама использует русский пример (`{"spec_markdown":"# Спецификация\n…"}`) — это часть проблемы (пример в нормативном документе тоже смещает язык).

**Требование пользователя (утверждено, не пересматривается):** язык контента сайта = «язык, на котором отвечает пользователь». Авто-детект из текста пользователя (промпт + ответы на вопросы), **без** изменения публичного API (`POST /projects` body не получает поле `language`), **без** миграции БД, **без** нового параметра в сигнатурах `agentN.py`. Override явным `locale` от клиента — вне MVP.

## Decision

**Авто-детект языка контента один раз Agent 2, фиксация маркером `**Content language:**` в начале `spec_markdown`, прокидывание downstream через уже передаваемый `spec_markdown` (новых колонок/полей/параметров НЕ вводится).**

1. **Детект — серверный, детерминированный, из исходного промпта ([ADR-028 §1–2](ADR-028-deterministic-source-prompt-language-detection.md)).** *(Исходная формулировка «детект — Agent 2 само-детект из всего ввода» отозвана ревизией ADR-028 как недетерминированная.)* Agent 2 получает язык **серверной директивой** (значение `generation_jobs.content_language`), пишет **весь** user-facing контент спеки на этом языке, и `spec_markdown` **обязан** начинаться явной строкой-маркером со значением из директивы:

   ```
   **Content language:** <название языка + BCP-47 код>
   ```
   Пример: `**Content language:** English (en)`.

   **Приоритет языка — см. [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md) §4 / [pipeline §03 → Язык/локализация](../modules/pipeline/03-architecture.md#языклокализация-контента-сайта--детерминированный-детект-adr-028-ревизует-adr-025) (единый нормативный источник): приоритет — ИСХОДНЫЙ промпт.** *(Исходная формулировка «приоритет ответов > промпт» отозвана ревизией ADR-028 — она порождала каскад прод-бага.)*

2. **Прокидывание — через `spec_markdown`.** Маркер живёт внутри `spec_markdown` (output Agent 2), который **уже** передаётся downstream (inline `spec_tz` / `spec_ref` в S3). Новый столбец/поле/параметр сигнатуры **не нужен**.

3. **Agent 3 (Builder).** Генерит **весь** видимый контент сайта на языке маркера `**Content language:**` из спеки и выставляет корневой `<html lang="<bcp-47>">`.

4. **Agent 4 (Fixer/Editor).** **Сохраняет** язык контента (не переключает, не переводит). Тот же маркер в той же неизменной спеке — источник языка и для fix-витка, и для edit-витка.

5. **Agent 1 (Interviewer).** Задаёт уточняющие вопросы **на языке оригинального промпта**. *(Ревизия [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md): язык подаётся Agent 1 серверной директивой из `content_language` — не «на усмотрение модели». Детерминированно.)*

6. **Дефолт «русский» убирается полностью.** Эталонные примеры в системных промтах **не должны содержать кириллицы / локаль-специфичных строк** (русских примеров output, термина `ТЗ`), чтобы few-shot не смещал язык вывода. Примеры в промтах — нейтральные/англоязычные плейсхолдеры. *(Правка самих `.txt`-промтов — зона backend; здесь фиксируется нормативное требование к их содержимому.)*

**Структура output Agent 2 НЕ меняется** — канонический ключ остаётся `spec_markdown` ([pipeline §I.1a](../modules/pipeline/03-architecture.md#i1a-канонические-output-схемы-агентов--single-source-of-truth-нормативно)); маркер — это первые символы **значения** строкового поля `spec_markdown`, а не новый top-level ключ. Валидатор `_validate_spec` не меняется по сигнатуре; добавляется лишь проверка наличия маркера в начале `spec_markdown` (зона backend, в рамках существующего валидатора).

## Consequences

**Плюсы.**
- Нулевое изменение **публичного контракта API**; фича в слое промтов + лёгкая валидация маркера. *(Схема БД: ревизия [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md) добавляет поле `content_language` ради детерминизма/crash-resume — публичный API при этом не меняется.)*
- Один источник детекта (теперь **серверный**, [ADR-028 §1–2](ADR-028-deterministic-source-prompt-language-detection.md)) — нет рассинхрона между агентами по языку.
- Маркер машинно-парсим (`**Content language:** <name> (<bcp47>)`) — Agent 3 извлекает BCP-47 для `<html lang>` детерминированно.
- Откат/смена политики не трогает API/БД.

**Минусы / риски.**
- ~~Детект — на стороне LLM (Agent 2), не детерминированный.~~ **Отозвано ревизией [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md):** именно недетерминизм LLM-детекта оказался прод-багом. Детект теперь **детерминированный серверный** (script-эвристика из исходного промпта, [ADR-028 §1](ADR-028-deterministic-source-prompt-language-detection.md)).
- Смешанный язык ввода разрешается правилом приоритета (**промпт**, [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md) §4) + fallback-правилом script-детекта ([ADR-028](ADR-028-deterministic-source-prompt-language-detection.md) §5). *(До ревизии ADR-028 здесь действовал отозванный приоритет «ответы > промпт».)*
- Явный locale-override от iOS-клиента (когда пользователь хочет сайт на языке, отличном от языка ввода) — **не покрыт**, вынесен в [Q-LOCALE-1](../99-open-questions.md#q-locale-1) (вне MVP, `blocks_sprint: none`).

**Требование к зависимостям.** Новых внешних библиотек/SDK не вводится (детект — собственная script-эвристика, чистый Python, [ADR-028 §1](ADR-028-deterministic-source-prompt-language-detection.md)). Правка `docs/02-tech-stack.md` не требуется.

## Alternatives

1. **Явный параметр `language`/`locale` в `POST /projects` (и проброс в сигнатуры агентов).** Отклонено: требует изменения публичного API, миграции (хранить выбранный язык), нового параметра в `agentN.py` — прямо против требования пользователя (без API/миграции/сигнатур). Также перекладывает выбор на клиента, тогда как продуктовое решение — «язык = язык пользователя» автоматически.
2. **Серверный детектор языка (`langdetect`/`fasttext` над промптом+ответами в backend) до вызова Agent 2.** Отклонялось для MVP: новая внешняя зависимость, отдельная точка рассинхрона, лишний слой. **ПЕРЕСМОТРЕНО ревизией [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md) (2026-06-11):** LLM-детект оказался **ненадёжным** (прод-баг: недетерминированные вопросы Agent 1 на тот же промпт). ADR-028 вводит серверный детерминированный детект — но **не библиотекой**, а **собственной узкой script-эвристикой** (доминирующий Unicode-script → язык; без новой зависимости, без правки `02-tech-stack.md`), и **из ИСХОДНОГО промпта** (не промпт+ответы), **до** Agent 1. Прежнее возражение «рассинхрон серверного детекта с пониманием Agent 2» снято: детект влияет лишь на **директиву**, финальную точность даёт LLM (см. [ADR-028 §1, §Alternatives п.2](ADR-028-deterministic-source-prompt-language-detection.md)). Библиотека `langdetect`/`fasttext` по-прежнему отклонена (шум на коротких промптах; для директивы избыточна).
3. **Детект в Agent 1 (Interviewer, LLM само-детект) с прокидыванием через отдельное поле.** Отклонено и в этой, и в ревизии [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md): LLM-само-детект недетерминирован (корень прод-бага). ADR-028 переносит детект **на сервер** (детерминированная script-эвристика) **до** Agent 1 и фиксирует результат в `content_language` — Agent 1 получает язык готовой серверной директивой, сам не детектит.
4. **Хранить выбранный язык в `generation_jobs` (новая колонка) и читать downstream из БД.** Отклонялось: миграция БД, дублирование источника. **ПЕРЕСМОТРЕНО ревизией [ADR-028](ADR-028-deterministic-source-prompt-language-detection.md):** колонка `generation_jobs.content_language` **вводится** — она нужна как **crash-устойчивый якорь** (язык фиксируется до Agent 1, переживает рестарт воркера между фазами) и как единый источник для Agent 1, который стартует **до** появления маркера в спеке. Маркер в спеке остаётся каналом для downstream Agent 3/4, но его значение приходит из `content_language` ([ADR-028 §2](ADR-028-deterministic-source-prompt-language-detection.md)).
