> **Прод-фикс ADR-017 (2026-06-08) — пустой экран path-сайта (`vite: not found` + потеря `--base`):** сгенерированные path-сайты открывались пустым экраном — HTML ссылался на ассеты по `/assets/...` (вместо `/s/{site_id}/assets/...`) → 404 за catch-all API-роутером → JS не исполнялся. Две причины: (1) голый `vite build` → `sh: vite: not found`; (2) «починка» Agent 4 на `npm run build` + хвостовой `--base` → npm не прокидывал base во vite → `base=/`. Фикс (не отдельный ADR — та же base-path-механика path-режима, уточнение [ADR-017 §Fix (2026-06-08)](ADR-017-path-based-site-routing.md#fix-2026-06-08--vite-not-found--потеря---base-через-npm-run-build-прод-инцидент)): воркер **нормализует** `build.command` к `npm install && npx vite build` (голый `vite build`/`npm run build` запрещены) и инжектит `--base=/s/{site_id}/` **в токен `npx vite build`** (не в хвост строки). Эталонный `build.command` Agent 3/4 — `npm install && npx vite build`; base задаёт воркер. Нормативно — [deploy §2A](modules/deploy/03-architecture.md#2a-path-based-routing-s-site_id-prod--site_routing_modepath-adr-017), [pipeline → Контракт output Agent 3](modules/pipeline/03-architecture.md#контракт-output-agent-3-полная-валидируемая-схема).
